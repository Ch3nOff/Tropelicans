# Tropelicans Roadmap

## Phase 1: Foundation

- Define project architecture and terminology.
- Create the CLI entry point.
- Add local configuration for agents, memory, workspace, and skills.
- Implement a minimal text-only runtime.

## Phase 2: Workspace runtime

- Add workspace indexing.
- Track file changes made by users and the AI.
- Add safe command execution policies.
- Store workspace summaries in project memory.

## Phase 3: Memory and knowledge

- Add short-term and long-term memory stores.
- Add importance scoring and expiration rules.
- Add incident memory for bugs, tests, and fixes.
- Build the second-brain knowledge view.

## Phase 4: Multi-agent orchestration

- Add specialized agent categories.
- Add lazy activation for inactive agents.
- Add agent-to-agent communication through the orchestrator.
- Add runtime resource usage reporting.

## Phase 5: Web customization

- Add first-time user onboarding.
- Add visual runtime configuration.
- Add agent, memory, skill, and workspace dashboards.
- Add user-customizable interface settings.
